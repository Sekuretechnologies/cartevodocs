## Objectif

Ce guide vous aide à intégrer rapidement les services essentiels de Cartevo.

## Étape 1 — Portefeuilles

Créer un portefeuille
```http
POST /wallets
Content-Type: application/json
Authorization: Bearer <token>

{
  "currency": "USD",
  "country_iso_code": "US"
}
```

Lister les portefeuilles
```http
GET /wallets
Authorization: Bearer <token>
```

Exemple cURL
```bash
curl -X GET "$BASE_URL/wallets" -H "Authorization: Bearer $API_KEY"
```

Réponse (200)
```json
[
  {
    "id": "wlt_123",
    "currency": "USD",
    "balance": 150.25,
    "created_at": "2025-10-01T12:00:00Z"
  }
]
```

## Étape 2 — Alimentation & Dépôt

Alimentation locale
```http
POST /wallets/fund
Content-Type: application/json
Authorization: Bearer <token>

{
  "userId": "<userId>",
  "walletId": "<walletId>",
  "amount": 10000,
  "phone": "+2376xxxxxxx",
  "operator": "ORANGE",
  "currency": "XAF"
}
```

Dépôt avec change
```http
POST /wallets/deposit
Content-Type: application/json
Authorization: Bearer <token>

{
  "sourceWallet": {
    "id": "<walletIdXAF>",
    "currency": "XAF",
    "amount": 10000,
    "feeAmount": 0,
    "totalAmount": 10000
  },
  "destinationWallet": {
    "id": "<walletIdUSD>",
    "currency": "USD",
    "amount": 15
  },
  "exchangeRate": {
    "rate": 650,
    "fromCurrency": "XAF",
    "toCurrency": "USD"
  }
}
```

Exemple cURL
```bash
curl -X POST "$BASE_URL/wallets/deposit" \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d @payload.json
```

## Étape 3 — Transactions

Lister les transactions d’un portefeuille
```http
GET /wallet/transactions/wallet/<walletId>
Authorization: Bearer <token>
```

Filtres utiles

- `status`: SUCCESS, PENDING, FAILED
- `type`: FUND, DEPOSIT, TRANSFER
- `page`, `limit`: pagination

## Étape 4 — Clients

Créer un client
```http
POST /customers
Content-Type: application/json
Authorization: Bearer <token>

{
  "first_name": "Jane",
  "last_name": "Doe",
  "email": "jane@example.com",
  "phone_number": "+2376xxxxxxx",
  "country_iso_code": "CM"
}
```

Conseils

- Validez les emails et indicatifs pays côté client
- Réconciliez les transactions clients via `GET /customers/:id/transactions`
